Password Security

Never store plain-text passwords — always hash with bcrypt or Argon2
Enforce minimum 8 characters with complexity requirements
Implement account lockout after 5 failed attempts
Use secure password reset with time-limited tokens
Never expose passwords in logs or error messages
// BAD - Never do this db.save({ password: userPassword }); // GOOD - Hash with bcrypt const hash = await bcrypt.hash(password, 12); db.save({ password: hash });

JWT & Session Security

Use short-lived JWT tokens (15 min access, 7 day refresh)
Store JWTs in httpOnly cookies, not localStorage
Implement token rotation and revocation lists
Always verify token signature on every request
Use strong secret keys (minimum 256-bit)
// BAD - localStorage is XSS-vulnerable localStorage.setItem('token', jwt); // GOOD - httpOnly cookie res.cookie('token', jwt, { httpOnly: true, secure: true, sameSite: 'strict', maxAge: 900000 });

Multi-Factor Authentication (MFA)

MFA adds a second layer of security beyond passwords. Even if a password is compromised, the attacker still cannot access the account without the second factor.

SMS OTP

One-time passwords sent via SMS. Simple but vulnerable to SIM swapping.

TOTP Apps

Google Authenticator, Authy. Time-based codes, much more secure than SMS.

Hardware Keys

YubiKey, FIDO2. The most secure MFA option, phishing-resistant.

Ethical Use Only: Penetration testing must only be performed on systems you own or have explicit written permission to test.

Reconnaissance

The first phase of pen testing — gather information about the target without active interaction.

Passive OSINT: Shodan, WHOIS, DNS enumeration
Active scanning: Nmap port & service discovery
Web fingerprinting: Wappalyzer, WhatWeb
Directory enumeration: Gobuster, Dirb
# Nmap basic scan nmap -sV -sC -p- target.com # Service version detection nmap -sV --version-intensity 5 target.com

Common Attack Vectors

SQL Injection — manipulate database queries
XSS — inject malicious scripts into web pages
CSRF — force users to perform unintended actions
Directory traversal — access restricted files
Broken authentication — bypass login systems
Insecure deserialization — execute arbitrary code

Network Security Basics

Always use HTTPS (TLS 1.2+) for all communications
Configure firewalls to deny all, allow only required ports
Use VPNs for remote access to internal networks
Implement network segmentation and VLANs
Monitor traffic with IDS/IPS systems
Regularly audit firewall rules and network access

API Security

Authenticate every API endpoint — no public endpoints by default
Implement rate limiting to prevent abuse
Validate and sanitize ALL input on the server side
Use HTTPS only — never expose APIs over HTTP
Implement proper CORS policies
Log and monitor all API access
